## Architecture Overview

### CLI Architecture

1. **Entry Point** (`my_unicorn/main.py`): Async main with uvloop integration
2. **CLI Layer** (`my_unicorn/cli/`): Argument parsing and command orchestration
3. **Command Handlers** (`my_unicorn/commands/`): Individual command implementations
4. **Services** (`my_unicorn/services/`): Business logic (progress, verification, cache)
5. **GitHub Integration** (`my_unicorn/github_client.py`): API client with caching
6. **Configuration** (`my_unicorn/config.py`): Catalog, Global and per-app configuration management

### Key Architectural Patterns

- **Command Pattern**: Each CLI command has a dedicated handler inheriting from `BaseCommandHandler`
- **Dependency Injection**: Shared dependencies (ConfigManager, AuthManager, UpdateManager) injected into handlers
- **Async Context Managers**: Progress tracking, session management, and resource cleanup
- **Service Layer**: Separate services for progress, verification, caching, and icon management
- **Data Structures**: TypedDict for JSON/API data, `@dataclass(slots=True, frozen=True)` for domain objects
